FIT9132 Introduction to Database

课程简介

  • 相较于一些偏理论或偏自学的课程，FIT9132 的 workshop 和 applied 课更强调通过课堂讲解与实际练习理解数据库知识。
  • 课程主要使用 Oracle 数据库，能学到不少实用内容。对我来说，最难的是 week 1 的 entity 识别与概念建模；这部分有点像阅读理解，很多同学也会在这里卡住。

主要内容

  • 概念模型
  • 关系模型
  • 规范化
  • 逻辑模型
  • 数据库的创建与修改
  • 插入、更新、删除（DML）与事务管理
  • SQL Part I：Basic
  • SQL Part II：Intermediate
  • SQL Part III：Advanced
  • 非关系型数据库（Non-Relational Databases）
  • 数据仓库及法律与伦理问题（Data Warehousing and Legal/Ethical Issues）

评估总结

  • Assignment 1：35%
  • Assignment 2：35%
  • TEST 1 in class：15%
  • TEST 2 in class：15%
  • 作业共两次，每次包含若干小任务和一个占比更高的大作业；课堂测试共两次，主要考察对练习内容的现场掌握情况。

学习建议

  • 先学好基础概念，尤其要理解每一种模型在表达什么、解决什么问题。
  • 多做练习，把课堂内容真正落实到建模和 SQL 操作中。
  • 积极参与课堂讨论，与同学交流建模思路。数据库题目很多时候没有一眼可见的答案，讨论能帮助你发现盲点。

常见难点

  • 概念模型、关系模型、规范化和逻辑模型构成了一个重要单元，也会直接影响大作业。识别 entity 并区分关系通常是最难的部分。
