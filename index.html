Monash AI 课程指南

本页面整理了 Monash 计算机科学与人工智能方向的基础课程经验，适合正在规划学习路线、准备实验、作业与考试的同学参考。

前言

我于 2026 年 2 月入学 Monash University，并在 IT 学院担任课程代表。本科阶段我学习的是商科，因此很理解跨专业同学，尤其是文科或商科背景的同学，在转向计算机时会遇到的困难。计算机不仅是知识体系的转换，更是一套思维方式的重建。

有同学说，理科在某种意义上也是一门哲学。毕竟 PhD 的全称是 Doctor of Philosophy。当你在后期面对模型、系统架构与抽象概念时，一些哲学式的追问反而可能帮助你打开思路。

因此，本文会参考北大的 CS 自学指南 和 上海交通大学飞跃手册，结合我在 Monash AI 专业的学习经历，分享如何入门计算机、如何备考相关课程，以及跨专业学习时可以采用的策略。

这份指南也想延续 CS 社区的开源精神，献给所有想学习 AI 或计算机的学弟学妹，也献给不断摸索中的我自己。希望它能给你一些启发、提醒和勇气。

课程目录

  • FIT9136 Introduction to Python — Python 基础课，也是跨入计算机专业的第一门编程语言。课程内容很新，也很有意义。建议在学好 Python 之后，将 C++ 和 Java 作为第二、第三门语言继续拓展。
  • FIT9132 Introduction to Database — 数据科学的重要基础课。Wong 教授是一位非常好的老师，毕业于 NUS。这门课帮助我补上了许多商科阶段没有真正理解的数据处理知识。
  • MAT9004 Math for AI and DS — 数学是我的弱项，这门课备考过程确实很痛苦，但该写的内容我都会尽量写清楚。它非常依赖高中数学基础；如果你学过高等数学，可以相对放松一些。
  • FIT5125 IT Research — IT 研究方法论。我个人觉得它有点像社会科学导论，也是后续了解 PhD 和学术研究的入门课。课程本身能学到不少内容，只是现实中有些同学并没有认真去上课。

如何申请 PhD

申请 PhD 需要先做风险评估。有些同学读硕士是为了进入腾讯、阿里等大厂，也有人希望未来申请 MIT 这类顶尖院校的 PhD candidate，当然后者也包括偶尔白日做梦的我自己。

但我并不建议大家盲目把 PhD 当成默认选项。它很大程度上取决于你的本科背景、科研经历和本科阶段导师资源，与 Monash 本身或授课型硕士的关系并没有想象中那么直接。如果本科阶段完全没有科研积累，仅仅希望通过授课型硕士直接转 PhD，现实难度会非常高。后续我会单独整理一篇 PhD 申请评估攻略。

如何找实习

找实习和申请 PhD 在底层逻辑上有一些相似之处：都需要长期积累，而不是临时冲刺。实习方向通常需要刷 LeetCode、做项目、维护 GitHub，并逐步建立自己对计算机行业的理解。

我推荐大家看一看 with Jiayi Weng talking。其中我最喜欢的一点是：计算机行业很强调能力和结果，因此你需要为自己建立一套评价体系，才能更清楚地知道如何学习、如何实践，以及如何在这个行业里持续成长。
